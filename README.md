## Описание проекта

Телеграм‑бот для бронирования переговорных комнат в офисе.  
Бот ведёт пользователя по сценарию «комната → дата → время», следит за пересечениями во времени, не даёт бронировать прошлое и при конфликте предлагает ближайший свободный слот.

**Основные особенности:**
- Полностью кнопочный UX (минимум ввода текста).
- Чёткая логика времени: только рабочие часы, защита от прошлого, строгий контроль пересечений.
- Мини‑аналитика использования переговорок (`/stats`).
- Удобный просмотр / отмена своих броней.
- Кнопка очистки истории чата и реферальная ссылка для коллег.

## Настройка

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN=ВАШ_TELEGRAM_BOT_TOKEN
```

> **Важно:** храните токен только в `.env`, не коммитьте его в репозиторий.

## Запуск

```bash

```

После запуска бот инициализирует базу данных, добавит три демонстрационные комнаты и начнёт получать апдейты через long‑polling.

## Функциональность и UX

- **`/start`**
  - Персональное приветствие по имени/нику пользователя.
  - Отправка главного меню с основными действиями.

- **Кнопка `🏢 Забронировать комнату`**
  - Показ списка комнат (название + вместимость).
  - При выборе комнаты — карточка с фото, описанием и кнопками выбора даты:
    - Сегодня.
    - Завтра.
    - Выбрать дату (ввод `DD.MM.YYYY`).
  - После выбора даты — список часовых слотов в рабочее время (9:00–18:00):
    - Для **сегодня** показываются только те слоты, которые ещё не закончились.
    - Текущее свободное окно помечается как  
      `09:00–10:00 🟢 (окно свободно до 10:00)`.
    - Занятые слоты отображаются с красным маркером `🔴` и недоступны для выбора.
  - При нажатии на свободный слот бот:
    - Проверяет пересечения в базе.
    - Подтверждает бронь красивым сообщением (жирное время, курсивная дата, пожелание).

- **Кнопка `📅 Мои бронирования` / команда `/my_bookings`**
  - Показывает все активные брони пользователя в читаемом виде.
  - Под каждым бронированием — кнопка отмены (`❌ …`).
  - При отмене бот обновляет сообщение и помечает бронь как отменённую.

- **Кнопка `📊 Статистика` / команда `/stats`**
  - Мини‑аналитика по всем броням:
    - Самая популярная комната (по количеству броней).
    - Самое загруженное время (час дня, когда больше всего броней).

- **Кнопка `🧹 Очистить историю`**
  - Пытается удалить последние сообщения бота и пользователя в чате (в рамках ограничений Telegram).
  - После очистки отправляет единое приветственное сообщение:
    - «Рад вас видеть вновь в нашей системе бронирования переговорных комнат!»
    - Сразу показывает выбор комнат и подсказывает про остальные функции в нижнем меню.

- **Кнопка `👥 Пригласить друга`**
  - Генерирует реферальную ссылку вида `https://t.me/<bot_username>?start=ref_<user_id>`.

## Архитектура проекта

- `bot.py` — точка входа:
  - создаёт `Bot` и `Dispatcher`,
  - инициализирует базу и сиды комнат,
  - подключает роутеры и запускает `start_polling`.

- `config.py` — загрузка настроек из `.env` (в первую очередь `BOT_TOKEN`).

- `database.py` — слой доступа к данным (SQLite, `aiosqlite`):
  - создание таблиц `rooms` и `bookings`,
  - сидирование комнат (`seed_rooms`),
  - выборка комнат и броней,
  - строгая проверка пересечений во времени,
  - функции для статистики и админской очистки броней.

- `services/booking_service.py` — бизнес‑логика работы со временем:
  - генерация часовых слотов с учётом текущей даты/времени,
  - пометка слотов как свободных/занятых с учётом уже существующих броней.

- `handlers/start.py` — обработчик `/start` и приветственный сценарий.

- `handlers/booking.py` — основная логика UX:
  - выбор комнаты, даты и слота,
  - подтверждение бронирования,
  - просмотр и отмена броней,
  - мини‑статистика,
  - очистка истории,
  - приглашение друзей.

- `keyboards.py` — фабрики клавиатур:
  - главное меню (reply‑клавиатура),
  - inline‑клавиатуры для комнат, дат, слотов, списка броней.

## Безопасность и валидация времени

- **Параметризованные SQL‑запросы**
  - Все запросы к SQLite используют `?`‑плейсхолдеры, например:
    - `SELECT * FROM rooms WHERE id = ?`
    - `INSERT INTO bookings (room_id, user_id, start_time, end_time) VALUES (?, ?, ?, ?)`
  - Это защищает от SQL‑инъекций.

- **Защита от бронирования в прошлом**
  - При выборе даты:
    - нельзя выбрать дату меньше сегодняшней — бот сразу отвечает ошибкой.
  - При генерации слотов на сегодня:
    - учитывается текущее время,
    - полностью прошедшие окна не попадают в список.

- **Проверка корректности даты**
  - При вводе пользовательской даты:
    - бот пытается распарсить формат `DD.MM.YYYY`,
    - при ошибке — формат `YYYY-MM-DD`,
    - при неудаче — понятное сообщение об ошибке и повторный ввод.

- **Контроль `end >= start`**
  - Слоты формируются как фиксированные интервалы `1 час`:
    - `end = start + 1 час`,
  - благодаря этому невозможно получить ситуацию `end < start`.

- **Строгий контроль пересечений времени**
  - Перед созданием брони выполняется SQL‑проверка:

    ```sql
    SELECT id
    FROM bookings
    WHERE room_id = :room_id
      AND start_time < :new_end
      AND end_time   > :new_start;
    ```

  - Если найдена запись, бот:
    - не создаёт новую бронь,
    - объясняет пользователю, в какой интервал уже занята комната:
      - «Комната занята с 10:00 до 11:00. Ваш интервал пересекается.»

- **Предложение альтернатив**
  - При конфликте времени бот ищет ближайший последующий свободный слот для этой комнаты и этой же даты.
  - Если такой слот есть, бот пишет, например:

    > Но есть свободная альтернатива: 11:00–12:00. Забронировать этот слот?

  - Это создаёт ощущение «умного» помощника, а не просто отказа.

- **Хранение токена**
  - Телеграм‑токен хранится только в `.env` и читается через `python-dotenv`.
  - В исходном коде и README он не захардкожен.

## Структура проекта

- `bot.py` — точка входа, инициализация бота, диспетчера и базы.
- `config.py` — загрузка настроек из `.env`.
- `database.py` — работа с SQLite: таблицы, CRUD по комнатам и бронированиям, статистика.
- `services/booking_service.py` — бизнес‑логика генерации слотов и проверки пересечений.
- `handlers/start.py` — обработка команды `/start`.
- `handlers/booking.py` — выбор комнаты, даты, времени, просмотр/отмена броней, статистика.
- `keyboards.py` — все inline‑ и reply‑клавиатуры.
- `images/` — изображения переговорных комнат.
